# 角色与用户管理指南

北极星企业版把“人”和“权限”拆成五层：

```text
用户身份 -> 部门/负责人 -> 治理角色与业务角色 -> Agent/设备信任 -> 每任务 Grant
```

角色只决定用户可以申请什么。真正执行仍需当前任务的 Grant，角色本身不能永久开放 MCP、网络、企业数据或业务写入。

## 老板第一次设置

1. 指定企业负责人和独立的安全管理员。
2. 创建部门，并为每个员工填写直属负责人和成本中心。
3. 给员工保留基础角色 `employee`，再按工作分配业务角色：运营管理、供应链管理、财务管理或产品管理。
4. 只有确有职责的人才分配用户管理员、部门负责人或审计员。
5. 每月查看临时角色到期、离职账号、无人负责的审批和异常成本。

## 业务负责人日常操作

- 新员工：先邀请，完成身份、部门、负责人、设备检查后再激活。
- 调岗：先冻结受影响任务并撤销旧 Grant/MCP 会话，再修改部门和角色。
- 临时支援：必须设置到期时间，到期自动失效。
- 离职：停用身份、终止 Agent 会话、撤销 Grant/MCP/凭据，并转交任务和产物。
- 批量导入：先 Dry Run；特权角色永远不能批量分配。

## 员工能看到什么

员工只看到自己的部门、角色、Agent 分配、当前 Grant 和费用归属。看见一个业务菜单不等于能读取全部数据；每次任务仍会检查组织、部门、项目、数据密级、设备、Agent 信任和预算。

## 角色速查

| 类型 | 角色 | 主要用途 | 不自动获得 |
|---|---|---|---|
| 治理 | 企业负责人 | 企业汇总、重大业务审批 | 平台管理、自己的 L4 审批 |
| 治理 | 部门负责人 | 部门用户、任务、预算 | 跨部门原始数据 |
| 治理 | 用户管理员 | 邀请、停用、维护身份 | 特权角色分配权 |
| 治理 | 安全管理员 | 设备、Agent、隔离、恢复 | 业务 L4 审批 |
| 治理 | 审计员 | 只读审计链 | 执行和审批 |
| 业务 | 运营管理 | Listing、广告、活动、售后 | 改价、改预算、上传 |
| 业务 | 供应链管理 | 采购、物流、库存 | 下单、付款、删除 |
| 业务 | 财务管理 | 利润、税务、合规、报表 | 申报、付款、外发 |
| 业务 | 产品管理 | 选品、市场、关键词、内容 | 上架、改价、发布 |

## 关键原则

- 发起人不能批准自己的请求。
- 执行人不能核验自己的结果。
- 企业负责人、安全管理员、用户管理员职责分离。
- 停用和撤销先于后台清理，运行时立即拒绝。
- 历史用户不物理删除，确保审批、回执和费用仍可追溯。
